详细描述

Appears to be some sort of interface between RMMapContents and markers.

Bug:
lots of arbitrary-appearing @synchronized blocks. Old mailing list traffic claims they're needed, but no one seems to know why. If the #set ivar needs to be guarded, should be done by @synchronized(self) and not @synchronized(sublayers). Maybe the guarding is needed because of Core Animation thread interactions.
